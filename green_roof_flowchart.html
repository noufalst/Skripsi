◈ SEGMENT 12 — Main Simulation Loop (dt=1s)
⟳ for step in range(N_steps) | N_steps = 604,800 (7 days)
INPUT Read Weather at step t
T_a_K, G_sol, RH, u, rain → convert rain to j_pr (kg/m²s)
T_in_current ← T1Ka/T3Kd NI data (dynamic, not 25°C!)
Seg 8 — Blok 1 🌿 Foliage Model → T_f(t)
compute_aerodynamic_resistance(u, plant)
→ r_a = f(H_f, u) CAM: r_a bigger (shorter plant)

compute_stomatal_resistance(G_sol, T_f, P_sat, P_a, θ)
→ r_stoma = r_stoma_min/LAI × f1(G_sol)×f2(T)×f3(VPD)×f4(θ)
CAM siang: ×12 (stomata hampir tutup)
CAM malam: ×0.5 (stomata terbuka)
C3 siang: tidak ada modifier

solve_foliage_temperature() [Eq.1,2,3]
→ R_f,net - h_conv×ΔT - h_eva×ΔP = 0
→ T_f_new (implicit scheme, direct solve)
Seg 9 — Blok 2a 🪨 Substrate Heat → T_g(z,t)
compute_substrate_properties(θ) [Eq. B.8-B.11]
→ ρcp_g, λ_g, k_θ, D_θ all change with θ

solve_substrate_heat(T_g, T_f, T_slab_top) [Eq.4,5,6,7]
→ PDE: (ρcp)∂T/∂t = ∂/∂z(λ∂T/∂z)
→ BC top: R_g,net - h_conv - h_eva [Eq.5]
→ BC bot: continuity with slab [Eq.7]
→ FVM → tridiagonal system
→ TDMA solve → T_g[0..106]
Seg 9 — Blok 2b 💧 Substrate Moisture → θ(z,t)
solve_substrate_moisture(θ, T_f, T_g[0]) [Eq.8,9,10,11]
→ Richards PDE: ∂θ/∂t = ∂/∂z(D_θ∂θ/∂z) + ∂k_θ/∂z
→ j_eva = j_eva_foliage + j_eva_substrat [Eq.10]
→ BC top: j_irrigation - j_eva [Eq.9]
→ BC bot: drainage if θ≥θ_sat [Eq.11]
→ FVM → TDMA solve → θ[0..106] + j_eva
Seg 10 — Blok 3 🏗️ Slab Conduction → T_s(z,t) + q_s_in
solve_slab_heat(T_s, T_g[-1], λ_g[-1]) [Eq.12,13,14]
→ Pure conduction, λ_s = const
→ BC top: continuity with substrate [Eq.13]
→ BC bot: h_in×(T_s[-1] - T_in_dynamic) [Eq.14]
→ FVM → TDMA solve → T_s[0..66]
→ q_s_in = h_in × (T_s[-1] - T_in)
q_s_in < 0 = heat sink ✅ | q_s_in > 0 = heat gain ⚠
SAVE Store Results (every 60s)
T_f T_g_top T_g_bot T_s_in θ_top/mid/bot q_s_in j_eva
604,800 steps → save 10,080 records/sim (memory efficient)
⟳ repeat until t = t_final
POST Compute Q_gain [Eq.18]
Q_gain = ∫ q_s_in dt (trapezoid rule)
Negative Q_gain = GR acts as heat sink ✅
Seg 11 ⚡ TDMA Solver (called 3×/step)
Thomas Algorithm: O(n) complexity vs O(n³) matrix inversion
Forward sweep → Back substitution
Used by: T_g (107) θ (107) T_s (67)
Seg 16 Run Simulation × 2
run_simulation(bromelia, ...) → results_CAM
run_simulation(wedelia, ...) → results_C3
Same weather data, different PlantParameters
Seg 13 LAI Calibration
Brent's method (scipy.optimize)
CAM: dual-target T_g_top + T_s_in
C3: single-target T_s_in only
~10-20 iterations to converge
Accuracy: LAI ± 0.005

Per value: T_s_in_max, Q_gain, ET
